- A film scene is a MEDIA PIPELINE, not a software application. It is rendered by EXTERNAL
  BINARIES that are pre-installed on the host: the ffmpeg CLI (assembly/encode), ComfyUI
  (image frames), Piper TTS (narration), FluidSynth (music). Your spec orchestrates those
  tools — it does NOT build a renderer.
- `dependencies` MUST be: ["none — renders via the ffmpeg CLI + ComfyUI + Piper/FluidSynth
  binaries (all external, pre-installed)"]. It is a HARD ERROR to list ANY Python image/video
  library: NEVER opencv-python, pillow, numpy, scipy, moviepy, imageio, or ffmpeg-python.
  Frames come from ComfyUI; compositing/encoding is done by the ffmpeg CLI directly.
- `build_system` MUST be "none (ffmpeg CLI)". `test_strategy` MUST be "ffprobe + frame_integrity".
- `file_structure` is MEDIA-SHAPED, not code-shaped. Use ONLY these:
    frames/        (ComfyUI output PNGs — directory placeholder)
    audio/         (Piper narration / FluidSynth music WAVs — directory placeholder)
    video/         (ffmpeg-encoded clip output — directory placeholder)
    shotlist.json  (optional shot manifest)
    render_scene.py  OR  render.sh   (OPTIONAL — a SINGLE script that shells the ffmpeg CLI)
  NEVER scaffold a code project: NO `src/` modules, NO `config.py`, NO `requirements.txt`,
  NO `__init__.py`, NO package layout. A title card or any scene is frames + audio + an
  ffmpeg encode — there is no application source to write.
- If a render script is included it MUST call the ffmpeg CLI via `subprocess.run(cmd, check=True)`
  (never a Python media library, never `print(cmd)`).
- Windows host: ffmpeg `drawtext` (fontconfig) and `geq=` in filter_complex are unavailable —
  note this in `risks` and prefer ComfyUI-generated text frames + `color=` solid backgrounds.
